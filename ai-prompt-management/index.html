Evolution

"잘 시킨다"의 진화

프롬프트에서 환경 설계까지, 의미가 넓어지고 있습니다

Prompt Engineering
어떤 말을 쓸까
"Think step by step", "You are an expert" 같은 마법의 단어를 찾던 시대였습니다.
프롬프트 마켓플레이스에서 잘 동작하는 프롬프트를 사고팔기도 했죠.
⬇
Context Engineering
어떤 정보를 줄까
컨텍스트 윈도우가 4K → 1M 토큰으로. AI가 프로젝트 전체를 읽고 파악합니다.
배경, 기술 스택, 기대 결과, 제약 조건까지 맥락을 빠짐없이 전달하는 거죠.
⬇
Harness Engineering
어떤 환경을 만들까
프롬프트가 "우회전해"라는 말이라면, 컨텍스트는 지도와 표지판.
하네스는 고삐, 안장, 울타리, 길 자체입니다.
CLAUDE.md, MCP 서버, 자동 린트 훅으로 환경이 규칙을 강제합니다.
이 셋은 대체 관계가 아니라 포함 관계입니다. 좋은 프롬프트 + 충분한 맥락 + 잘 설계된 환경.

Summary

핵심 요약

1
AI는 시키지 않으면 아무것도 안 합니다.
"잘 시키는 능력"이 AI 시대의 핵심 역량입니다.
2
잘 시키는 방법은 프롬프트 → 컨텍스트 → 하네스로 진화하고 있습니다.
어떤 말을 쓸까 → 어떤 정보를 줄까 → 어떤 환경을 만들까.
3
프롬프트가 많아지면 관리가 필요합니다.
관리를 잘하면 동시에 여러 프로젝트를 굴리는 "멀티 서클"이 가능해집니다.
지금 나의 프롬프트 관리는 어떻게 하고 있나요?
