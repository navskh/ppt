The Paradox

AI 시대의 역설

코딩은 AI가 해주는데, 왜 개발은 더 복잡해졌을까?

⚡
AI가 바꾼 것
Claude, GPT, Gemini... AI가 코드를 대신 짜주는 시대.
개발의 실행 속도는 폭발적으로 빨라졌다.
🧠
바뀌지 않은 것
하지만 "무엇을 만들지" 결정하는 건 여전히 사람의 몫.
아이디어 정리, 우선순위, 작업 분해는 AI가 못 한다.
🎯
진짜 병목은 "정리"
AI에게 좋은 결과를 받으려면, 좋은 프롬프트가 필요하다.
좋은 프롬프트는 잘 정리된 생각에서 나온다.
결국 생각을 구조화하는 과정이 개발의 핵심이 되었다.
Limitations

기존 도구들의 한계

좋은 도구들이지만, AI 시대의 개발 워크플로우에는 맞지 않는다

📝
메모 앱
Notion, Obsidian, 메모장
❌ 적기엔 좋지만 실행으로 연결이 안 됨
❌ 아이디어가 쌓이기만 하고 정체
❌ AI 에이전트와 연동 불가
📋
프로젝트 관리
Jira, Linear, Trello
❌ 팀 협업 도구라 혼자 쓰기엔 무거움
❌ 브레인스토밍 → 태스크 전환이 단절
❌ AI 프롬프트 관리 개념 없음
🤖
AI 도구
Claude, ChatGPT, Cursor
❌ 대화가 끝나면 맥락이 사라짐
❌ 매번 처음부터 상황 설명 반복
❌ 프롬프트가 체계적으로 관리 안 됨
"아이디어를 적는 도구"와 "실행하는 도구" 사이에
연결 고리가 없다
Philosophy

IM의 철학

프롬프트가 곧 산출물이다

1
생각을 자유롭게, 정리는 도구가
브레인스토밍은 구조에 얽매이지 않고 자유롭게. 그 혼란스러운 생각을 서브 프로젝트와 태스크로 정리하는 건 IM의 역할입니다. 떠오를 때 적고, 나중에 정리하세요.
2
프롬프트는 일회성이 아니라 자산이다
AI에게 던진 프롬프트는 사라지지 않습니다. 태스크마다 프롬프트를 저장하고, 다듬고, 재사용합니다. 잘 만든 프롬프트 하나가 수시간의 코딩을 대체합니다.
3
사람이 지시하고, AI가 실행한다
프롬프트를 "제출"하면 AI 에이전트가 자동으로 실행합니다. MCP 서버와 Watch 모드로 Claude, Gemini, Codex가 태스크를 직접 수행합니다. 당신은 감독만 하세요.

Automation

Watch 모드 & Git 동기화

제출된 태스크를 자동 실행하고, 여러 PC에서 데이터를 동기화합니다

👀
Watch 모드
제출된 태스크를 AI CLI가 자동으로 실행합니다. 실시간 스트리밍 출력으로 진행 상황을 확인할 수 있습니다.
im watch # 자동 실행
im watch --project <id> # 특정 프로젝트
im watch --dry-run # 미리보기
🔄
PC 간 동기화
Git 리포지토리를 통해 여러 PC에서 데이터를 동기화합니다. GitHub CLI로 리포 자동 생성도 지원합니다.
# 첫 번째 PC
im sync init # 초기화
im sync push # 내보내기
# 다른 PC
im sync init # 연결
im sync pull # 가져오기

Quick Start

지금 바로 시작하세요

# 1. 설치
npm install -g idea-manager
# 2. 실행
im start
# 3. MCP 서버 연결 (선택)
claude mcp add idea-manager -- npx -y idea-manager mcp
# 4. 자동 실행 (선택)
im watch
