The Limits

비개발자 바이브 코딩의 벽

혼자 바이브 코딩하면 부딪히는 4가지 한계

🧠

컨텍스트 부재

AI는 우리 회사의 시스템 구조, API, DB 스키마를 모릅니다. 매번 처음부터 설명해야 합니다.

🎨

UI 일관성 깨짐

AI가 MUI, Ant Design 등 제각각인 라이브러리를 사용해 디자인 시스템이 무너집니다.

🔗

통합 실패

API 호출 규칙, 인증 토큰 처리, 에러 핸들링을 모르면 다른 시스템과 연동이 안 됩니다.

🚀

배포 불가

Docker, Kubernetes, Helm... 인프라 지식 없이는 만든 코드를 서비스에 올릴 수 없습니다.

The Solution

JABIS Maker

바이브 코딩의 한계를 해결하는 플랫폼

🌐

웹 브라우저만 있으면 됨

개발 환경 설치 불필요. 브라우저에서 바로 접속해서 코딩 시작.

📚

전체 문서 자동 주입

22개 프로젝트의 아키텍처, API, DB 스키마가 AI에게 자동 전달.

📏

MakeBible 규칙 내장

디자인 시스템, API 규칙, 코딩 컨벤션을 AI가 자동으로 준수.

🚀

자동 배포

git push만 하면 ArgoCD가 K8s에 자동 배포. 인프라 지식 불필요.

Getting Started

첫 대화까지

5단계로 시작하세요

1
로그인

브라우저에서 JABIS Maker에 접속, 발급받은 계정으로 로그인

2
프로젝트 선택

왼쪽 사이드바에서 작업할 프로젝트를 찾고, + 버튼으로 세션 생성

3
세션 연결 확인

터미널에 "세션 연결됨" 메시지가 나타나면 준비 완료

4
프롬프트 입력

하단 입력창에 하고 싶은 작업을 입력하고 Send 클릭 (또는 Enter)

5
응답 확인

파란 카드 = 도구 사용, 초록 카드 = 실행 결과, 일반 텍스트 = AI 설명

MakeBible

바이브 코딩의 대전제

이 규칙을 지켜야 통합이 됩니다

✓ 반드시 이렇게

UI 컴포넌트는 @jabis/ui만 사용
아이콘은 lucide-react만 사용
스타일은 Tailwind CSS 클래스만
색상은 시맨틱 토큰 bg-primary
API는 GET과 POST만
상태관리는 Zustand만

✗ 절대 금지

MUI, Ant Design, react-bootstrap
Font Awesome, 다른 아이콘 라이브러리
인라인 스타일 style={{}}
직접 색상 #FF0000
PUT, PATCH, DELETE 메소드
Redux, MobX, Context 남용
AI가 이 규칙을 이미 알고 있지만, 여러분이 알고 있으면 더 정확한 결과를 얻을 수 있습니다.
Tips & Pitfalls

흔한 실수 & 꿀팁

미리 알면 시행착오를 줄일 수 있습니다

⚠️
MUI/Ant Design 사용 → 통합 깨짐

다른 UI 라이브러리를 쓰면 JABIS 디자인 시스템과 충돌합니다. 반드시 @jabis/ui만 사용하세요.

🚨
git push = 즉시 배포

ArgoCD가 자동으로 K8s에 배포합니다. push 전에 반드시 코드를 확인하세요.

💡
미리보기는 pnpm build:preview

pnpm build로 하면 빈 흰 화면이 나올 수 있습니다. 반드시 build:preview를 사용하세요.

🔗
jabis-common 수정 후 consumer sync 필수

공통 라이브러리 수정 후 모든 소비 프로젝트에서 submodule 업데이트를 해야 합니다.
